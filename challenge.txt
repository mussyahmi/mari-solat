🌙 Cabaran Qada Bulanan — Spesifikasi Akhir

Tujuan:
Cabaran bulanan bertujuan membantu pengguna kekal konsisten menunaikan solat qada dengan menggunakan Nama Samaran (anonim). Pengguna log solat harian, mengekalkan streak, melihat kedudukan bulanan, berinteraksi dalam chat awam, dan sistem mengurus tingkah laku tidak sesuai secara komuniti, tanpa campur tangan admin harian.

Peraturan Utama
Cabaran berlangsung dari 1 haribulan hingga akhir bulan.
1 log qada per hari → kira 1 hari sahaja untuk streak, walaupun log lebih dari 1 solat.
Terlepas 2 hari berturut-turut → streak semasa kembali 0, tetapi pengguna masih aktif.
Nama Samaran dijana automatik selepas pendaftaran, boleh ditukar bila-bila masa.
Pengguna auto-enroll selepas daftar, tanpa perlu klik join.
Tidak boleh log untuk hari lepas / backfill.
Leaderboard akhir bulan menunjukkan:
Streak Terpanjang
Jumlah Qada Diselesaikan
Hari Aktif
Pengguna dengan streak / jumlah qada sama → sama ranking.
Pengguna yang selesai qada awal → tampil dengan lencana “Qada Selesai”.
Leaderboard boleh ditapis mengikut jenis kedudukan dan bulan.
Pengguna tanpa log → streak = 0, disembunyi sehingga log pertama.
Chat awam:
Pesan lama auto-deleted selepas tempoh tertentu (>30 hari).
Chat adalah Nama Samaran sahaja, tiada nama sebenar.
Pengguna boleh lapor mesej tidak sesuai, had laporan = 3 per hari.
Hanya satu laporan per pengguna dikira, boleh undo report.
Mesej dilaporkan cukup threshold → auto-hide.
Pengguna mesej tersembunyi → mute sementara, masa mute meningkat jika berulang tetapi reset jika behave, hanya untuk mesej masa depan.
Chat adalah untuk hormat-menghormati, jangan salah guna laporan.
✅ Keperluan Fungsi (User / System Should Be Able)
1) Akaun & Nama Samaran
Pengguna boleh daftar akaun.
Sistem menjana Nama Samaran automatik.
Pengguna boleh tukar Nama Samaran bila-bila masa.
Sistem papar Nama Samaran sahaja, bukan nama sebenar.
Sistem sync log & streak di semua peranti (menggunakan Firebase Auth).
2) Penyertaan Cabaran (Auto-Enroll)
Sistem auto-enroll pengguna selepas daftar.
Streak mula dari 1, walau daftar pertengahan bulan.
Sistem reset streak semasa ke 0 selepas 2 hari terlepas berturut-turut.
Streak sambung semula jika log selepas 1 hari terlepas.
Pengguna tanpa log → streak = 0, disembunyi sehingga log pertama.
Pengguna selesai qada awal → streak berhenti tetapi boleh naikkan streak tambahan selagi tidak terlepas 2 hari.
Streak dan Hari Aktif reset setiap bulan.
3) Log Solat Qada
Pengguna boleh log qada harian.
Log lebih dari 1 solat per hari → kira 1 hari sahaja untuk streak.
Tidak boleh log hari lepas / backfill.
Pengguna boleh edit log hari sama.
Sistem simpan semua log untuk laporan.
4) Pengiraan Streak & Aktiviti
Sistem kira streak semasa setiap pengguna.
Sistem reset streak ke 0 selepas 2 hari terlepas berturut-turut.
Sistem kira streak tertinggi bulan ini.
Sistem kira Hari Aktif (hari dengan ≥1 log qada).
Sistem kira jumlah qada diselesaikan bulan ini.
Pengguna selesai qada awal → streak berhenti tetapi boleh naikkan streak tambahan.
Pengguna mid-month → streak mula dari 1 pada hari pertama log.
Streak boleh terus bertambah sehingga akhir bulan selagi tidak terlepas 2 hari.
5) Leaderboard & Keputusan Bulanan
Sistem papar leaderboard akhir bulan.
Pengguna boleh toggle jenis kedudukan:
Streak Terpanjang
Jumlah Qada
Hari Aktif
Pengguna boleh pilih bulan.
Pengguna tanpa log → disembunyi sehingga log pertama.
Pengguna selesai awal → tampil dengan lencana “Qada Selesai”.
Streak / jumlah qada sama → ranking sama.
Sistem highlight pengguna semasa walaupun bukan top N.
Sistem papar Nama Samaran sahaja.
6) Notifikasi & Peringatan
Sistem hantar push notification harian untuk log qada, walaupun pengguna tidak buka app.
Sistem boleh peringat pengguna jika hampir putus streak.
Sistem boleh notifikasi cabaran bulan baru pada 1 haribulan.
7) Chat Awam & Moderasi Komuniti
Pengguna boleh hantar & baca mesej chat awam.
Sistem padam mesej lama automatik (>30 hari), hilang sepenuhnya.
Sistem papar Nama Samaran sahaja.
Pengguna boleh lapor mesej tidak sesuai, had laporan = 3 per hari.
Hanya satu laporan per pengguna dikira, boleh undo report.
Mesej dilaporkan cukup threshold → auto-hide.
Pengguna mesej tersembunyi → mute sementara, masa mute meningkat jika berulang tetapi reset jika behave, hanya untuk mesej masa depan.
Chat adalah untuk hormat-menghormati, jangan salah guna laporan.
Semua moderasi berlaku secara automatik, tiada campur tangan admin harian.
8) Admin / Backend (Minima)
Admin boleh lihat peserta & log qada (untuk troubleshooting).
Admin boleh eksport leaderboard.
Admin boleh ubah streak / log secara manual jika perlu.
Admin boleh urus tetapan chat / threshold / storan, tetapi tidak terlibat dalam moderasi harian.
